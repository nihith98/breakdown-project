# Getting Started with breakDown

How to navigate this project documentation.

## For Different Audiences

### 👤 Job Interviewers / Resume Reviewers
**Start here** to understand the project scope:
1. Read **[README.md](../README.md)** — 2 min overview
2. Explore **[PROJECT_OVERVIEW.md](./PROJECT_OVERVIEW.md)** — 10 min full vision
3. Check **[COMPLETED_WORK.md](./COMPLETED_WORK.md)** — 5 min what's done
4. Browse **[REPOSITORIES.md](./REPOSITORIES.md)** — Where the code lives

**Time**: ~20 minutes to understand the full scope

---

### 👨‍💻 Backend Developers
**Start here** to understand how to contribute:
1. Read **[ARCHITECTURE.md](./ARCHITECTURE.md)** — Module structure
2. Review **[CODING_CONVENTIONS.md](./CODING_CONVENTIONS.md)** — Java patterns
3. Clone repos from **[REPOSITORIES.md](./REPOSITORIES.md)**
4. Follow build order (see REPOSITORIES.md)
5. Reference **[PROJECT_OVERVIEW.md](./PROJECT_OVERVIEW.md)** for details

**Setup**: Clone repos → Install Maven → Build in order → Read CLAUDE.md for conventions

---

### 🎨 Frontend / Mobile Developers
**Start here** to understand design requirements:
1. Review **[DESIGN_SYSTEM.md](./DESIGN_SYSTEM.md)** — Colors, typography, components
2. Read **[branding/frontend-implementation-guide.md](../branding/frontend-implementation-guide.md)** — Complete specs
3. Check **[REPOSITORIES.md](./REPOSITORIES.md)** for frontend repos
4. Reference **[PROJECT_OVERVIEW.md](./PROJECT_OVERVIEW.md)** for platform requirements

**Key Files**: 
- Colors, typography, spacing specs
- Component state tables (buttons, inputs, cards)
- Responsive breakpoints
- Accessibility requirements

---

### 📊 Product Managers / Stakeholders
**Start here** for product vision:
1. Read **[branding/branding-philosophy.md](../branding/branding-philosophy.md)** — User archetypes, feature philosophy
2. Explore **[PROJECT_OVERVIEW.md](./PROJECT_OVERVIEW.md)** — Full feature set
3. Check **[COMPLETED_WORK.md](./COMPLETED_WORK.md)** — Status and timeline
4. Review feature stories in **[PROJECT_OVERVIEW.md](./PROJECT_OVERVIEW.md)** — What we're building

**Key Insights**: Privacy-first, family settlements, optimal settlement algorithm, multi-platform
